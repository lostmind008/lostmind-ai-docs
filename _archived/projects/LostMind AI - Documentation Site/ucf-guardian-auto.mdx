# Command: /ucf-guardian-auto

## Your Role: Guardian Automation Manager

This command configures automatic refresh settings for The Guardian context system, allowing users to set up periodic context updates without manual intervention.

## Your Workflow

1.  **Parse Arguments:** Handle user preferences for auto-refresh:
    - **No arguments**: Show current auto-refresh status
    - **"enable"**: Enable auto-refresh with default settings
    - **"disable"**: Disable auto-refresh
    - **Frequency options**: "manual", "hourly", "daily", "weekly"

2.  **Execute Configuration:** Run the Guardian wrapper with appropriate settings.
    ```bash
    ENGINE_PATH="$HOME/.claude/lostmind_ai_engine"
    if [ ! -f "$ENGINE_PATH/guardian_wrapper.py" ]; then
        echo "‚ùå LostMind AI engine not found. Please run the framework installer."
        exit 1
    fi

    # Show current status
    python3 "$ENGINE_PATH/guardian_wrapper.py" auto

    # Enable auto-refresh
    python3 "$ENGINE_PATH/guardian_wrapper.py" auto --enable --frequency daily

    # Disable auto-refresh
    python3 "$ENGINE_PATH/guardian_wrapper.py" auto --disable
    ```

3.  **Report Results:** Confirm the configuration changes and explain implications
4.  **Provide Guidance:** Suggest optimal settings based on project activity

## Usage Examples

### Enable Auto-Refresh
```bash
/ucf-guardian-auto enable daily
```
- Enables automatic context refresh once per day
- Perfect for active development projects

### Disable Auto-Refresh  
```bash
/ucf-guardian-auto disable
```
- Disables automatic refresh
- Returns to manual-only refresh mode

### Check Current Settings
```bash
/ucf-guardian-auto
```
- Shows current auto-refresh configuration
- Displays last auto-refresh activity

## Frequency Options

- **manual** (default): No automatic refresh, user-triggered only
- **hourly**: Refresh context every hour (high activity projects)
- **daily**: Refresh context once per day (normal development)
- **weekly**: Refresh context weekly (maintenance mode)
