### üìã **Required Environment Variables**
**Critical for basic functionality**:
- `DATABASE_URL`: PostgreSQL for session storage
- `SESSION_SECRET`: Secure session encryption key
- `REPLIT_DOMAINS`: Your app domain (localhost:5000 for dev)
- `OPENAI_API_KEY` OR `GEMINI_API_KEY`: AI chat responses

**Required for payments (if testing Stripe)**:
- `STRIPE_SECRET_KEY`: Stripe test key
- `STRIPE_PRICE_ID_PRO_MONTHLY`: Must be created in Stripe dashboard

**See .env.example for complete list with explanations**

## üîê Authentication Flow

1. User clicks "Sign In" or "Get Started"
2. Redirected to `/api/login` (Express route)
3. Replit OAuth flow handles authentication
4. User redirected back to `/api/callback`
5. Session created and user redirected to dashboard
6. Frontend uses `useAuth` hook to check authentication status

## üíæ Data Flow

### Scraping Workflow
1. User submits URL via scraping form
2. Job created in database with "queued" status
3. Request sent to FastAPI worker service
4. Worker processes website and extracts content
5. Results stored as documents in knowledge base
6. Job status updated to "done" or "fail"

### Chat Workflow
1. User selects documents for context
2. Chat message sent to backend API
3. Message stored in conversation history
4. AI model generates response using document context
5. Response stored and sent back to frontend
6. Credits deducted from user account
