## Part 1: Database Setup

### PostgreSQL with Row Level Security (RLS)

#### Option A: Supabase (Recommended for MVP)

**Dashboard**: https://supabase.com/dashboard

**Steps**:

1. Sign up/login at [Supabase Dashboard](https://supabase.com/dashboard)
2. Click **"New Project"**
3. Choose organization ‚Üí Enter project name: `lostmind-ai-dev`
4. Select region: **US East** (closest to Vercel)
5. Set database password (save this!)
6. Click **"Create new project"** (wait 2-3 minutes)

**Get Connection Strings**:

1. Go to **Settings** ‚Üí **Database**
2. Scroll to **Connection string**
3. Copy **"Postgres"** connection string
4. Replace `[YOUR-PASSWORD]` with your actual password

```bash
# Add to .env.local
DATABASE_URL="postgresql://postgres:[YOUR-PASSWORD]@db.[PROJECT-REF].supabase.co:5432/postgres"
DIRECT_URL="postgresql://postgres:[YOUR-PASSWORD]@db.[PROJECT-REF].supabase.co:5432/postgres"
```

#### Option B: Neon (Alternative)

**Dashboard**: https://console.neon.tech/app/projects

**Steps**:

1. Sign up at [Neon Console](https://console.neon.tech)
2. Click **"Create a project"**
3. Enter name: `lostmind-ai`
4. Select region: **US East (Virginia)**
5. Click **"Create project"**

**Get Connection String**:

1. In project dashboard, click **"Connection string"**
2. Copy the **"Pooled connection"** string

```bash
# Add to .env.local
DATABASE_URL="postgresql://[user]:[password]@[endpoint]/[dbname]?sslmode=require"
DIRECT_URL="postgresql://[user]:[password]@[endpoint]/[dbname]?sslmode=require&pgbouncer=true&connect_timeout=10"
```

**Verification**:

```bash
# Test connection
pnpm run db:push
# Should see "Your database is now in sync with your schema"
```

### Common Issues & Solutions

#### "Service Not Found" Errors

- Verify `AI_COMPUTE_SERVICE_URL` is set correctly
- Check that ai-compute service is running on port 8000
- Ensure frontend apps use `NEXT_PUBLIC_AI_SERVICE_URL`

- Add frontend URLs to `CORS_ALLOWED_ORIGINS` in ai-compute service
- Include all development ports: 3000, 3001, 3002

#### Port Conflicts

- Follow the port assignments from docker-compose.yml
- Use different ports if conflicts occur with other local services

## üö® Common Issues & Solutions

### "Missing NEXTAUTH_SECRET"

```bash
# Generate a new secret
openssl rand -base64 32
# Add to .env.local as NEXTAUTH_SECRET
```

### "Google OAuth Error"

- Check redirect URIs in Google Console
- Ensure `NEXTAUTH_URL` matches your domain
- Verify `GOOGLE_CLIENT_ID` and `GOOGLE_CLIENT_SECRET`

### "Database Connection Error"

- Check `DATABASE_URL` format
- Ensure database is running (for local PostgreSQL)
- Verify network access (for cloud databases)

### "Stripe Webhook Failed"

- Check webhook URL is accessible
- Verify `STRIPE_WEBHOOK_SECRET` matches Stripe dashboard
- Test with Stripe CLI: `stripe listen --forward-to localhost:3000/api/webhooks/stripe`

### "Gemini API Error"

- Verify API key is correct
- Check if Generative AI API is enabled in Google Cloud Console
- Ensure project ID matches `coral-muse-469919-m0`

### "AI Service Connection Failed"

- Check if ai-compute service is running: `curl http://localhost:8000/health`
- Verify `AI_COMPUTE_SERVICE_URL` is set in frontend apps
- Ensure CORS is configured: add frontend URLs to `CORS_ALLOWED_ORIGINS`
- Check docker-compose services: `docker-compose ps`

### "Port Already in Use"

- Check what's using the port: `lsof -i :8000`
- Stop conflicting services or use different ports
- Update port assignments in both docker-compose.yml and environment variables
- Restart all services: `docker-compose down && docker-compose up`

### "Service Not Found" (Microservices)

- Verify service discovery URLs are correct
- Check that all services started: `pnpm run dev` shows all apps
- Test individual service health endpoints
- Ensure `NEXT_PUBLIC_` prefix for client-side variables
